# らしさ を詰める

## 現状の構造

LLMに渡るキャラクター情報：

| 情報 | 具体性 |
|------|--------|
| MBTI | 16タイプのラベル |
| 5特性値（sociability等） | 1-5の数値 |
| 語彙プリセット（UUID参照） | カスタマイズ可 |
| 一人称 | 文字列 |
| 興味タグ | 最大20個 |
| 職業プリセット | UUID参照 |

**問題の所在**：数値やラベルが実際のセリフにどう変換されるか、の部分。

---

## 詰め方の順番

### Step 1: 現プロンプトを読む

`ConversationBrief` → LLMプロンプト変換の実装を特定する。
キャラクター情報がどんな文字列としてプロンプトに入っているかを確認する。

```
"speakerAppraisal" "MBTI" "personality" などでgrepする
```

### Step 2: 最も弱いシグナルを特定する

- MBTI単体はLLMには弱い（汎用知識に依存しすぎる）
- 数値5段階は抽象的すぎて差が出にくい
- 語彙プリセットが実質的に最大のレバーになりうる

### Step 3: 2キャラで同じイベントを流してみる

意図的に正反対の特性を持つキャラ同士に同じ `experience_event` を与えて会話生成し、
セリフの差が出ているか目視確認する。

### Step 4: 弱い部分を補強する

差が出ていない場合の選択肢：

**A. プロンプトに特性の「行動翻訳」を追加する**

```
stubbornness: 5 → 「自分の意見を簡単に変えず、相手の反論に対して理由を述べて返す」
sociability: 1  → 「会話は短く、必要な情報だけ伝えようとする」
```

**B. 語彙プリセットに例文を持たせる**

抽象的なスタイル名ではなく、実際の語尾・フレーズパターンを記述する。

**C. appraisalをキャラの声で書く**

現状の `appraisal` が客観的すぎるなら、特性値に応じて文体を変える。

---

## 最初にやること

1. `ConversationBrief` を組み立てている箇所のコードを読む
2. そこでキャラ情報がどう文字列化されているか確認する
3. 出力結果（実際のセリフ）と見比べて「どこで個性が潰れているか」を特定する

それさえわかれば、打ち手は自然に決まる。
