# 07. UI・画面設計

---

## 7.1 全体構造

Reladenは3D回転する「部屋」を中心に、**ホーム（front）**・**管理室（right）**・**日報（left）**の3面構成。Framer MotionによるCSS 3D transformsでルーム間を遷移し、管理室・日報では「机」が全面に展開する演出がある。

**共通レイアウト**: Header / RoomStage / DetailLayer（会話ログ）/ ConsultDetailLayer（相談ログ）/ RealtimeSubscriber

---

## 7.2 画面一覧

### ホーム画面（`/home`）

住人たちの「今」を一覧するメインのダッシュボード。3D部屋の正面（front）。空と床のイラスト背景上にガラスパネルが浮かぶ。

| 構成要素 | 内容 |
|---|---|
| 通知エリア | 会話パネル（未読件数 + 最新4件）と相談パネル（未読件数 + 最新2件）の分離表示 |
| 住人ステータスリスト | 各住人の現在状態をアイコン付きで表示 |
| 天気表示 | ワールド天気情報とAI生成コメント |
| 時計表示 | リアルタイム時刻と日付 |
| 会話/相談カード | 進行中の会話・相談をカード形式で表示 |

### 管理室トップ（`/office`）

住人管理メニューへのハブ画面。机パネル上にボタンリンクが並ぶ。

- 住人管理: 住人一覧 / 新規住人登録 / プリセット管理
- 管理室メニュー: 設定 / 遊び方 / 仕様について

### 住人一覧（`/office/residents`）

登録済みの住人をリスト表示し、CRUD操作を行う画面。

### 住人詳細（`/office/residents/[id]`）

住人のプロフィール・関係・変化を閲覧するタブ付き詳細画面。

| タブ | 内容 |
|---|---|
| プロフィール | 基本情報・会話・睡眠・パーソナリティ |
| 関係 | 関連住人一覧（相手名・関係種別・印象） |
| 最近の変化 | 関係/印象/好感度/信頼度の変化を時系列リストで表示（設計決定事項は後述） |

#### 「最近の変化」タブ（設計決定事項）

会話詳細フッターに表示される system メッセージと同じ表現の変化テキストを、該当住民でフィルタして一覧表示するシンプルなリスト。

- **表示例:** `3/18 20:10　山田 から 田中 への好感度が上昇しました。`
- **表示件数:** 最新15件、ページネーションなし
- **日時:** 各行に日付＋時刻を表示
- **日報リンク:** 各行に該当イベントの日報画面へ遷移するリンクボタン
- **データソース:** `events` テーブルから該当住民が `participants` に含まれるイベントを取得し、SYSTEM行をパース
- **パースロジック:** `parseSingleSystemLine` を共通ユーティリティとして切り出し、会話詳細・日報と共用
- **対象変化:** 好感度（上昇/下降）、印象変化、関係遷移、信頼度変化 |

### 住人編集（`/office/residents/[id]/edit`）/ 新規作成（`/office/new`）

住人の全属性を編集するフォーム画面。共通の `ResidentForm` コンポーネントを使用。

### 関係詳細（`/office/relations/[id]`）

2人の住人間の関係性を可視化する画面。

- 関係種別カード（中央配置）
- 左右対比パネル（A→B / B→A）: 印象・好感度バー・呼び方
- 直近の関わり: 最新5件の会話イベントリスト

### プリセット管理（`/office/presets`）

話し方・職業・一人称のプリセットを管理する3タブ構成。LLM自動生成ボタン付き。

### 日報画面（`/reports`）

過去の会話・相談イベントを時系列で閲覧するレポート画面。3D部屋の左面（left）。

- フィルタ: 日付・キャラクター・変化種別
- イベントリスト: 日付見出し付き時系列表示、変化チップ（好感度/印象/関係/信頼度）
- 相談イベント: **回答済み（`selectedChoiceId` あり）** のみ表示
- スライドインオーバーレイで詳細パネルを表示

### 設定画面（`/settings`）

データ管理（アカウント・同期・削除）、アクセシビリティ（テーマ・フォントサイズ・軽量設定）、ゲーム情報。

### 遊び方（`/playguide`）

ゲームの基本的なプレイガイドの静的コンテンツ。

### 仕様について（`/specs`）

プレイヤー向けの詳細仕様説明ページ。「遊び方」の詳しい版。（実装予定）

### 相談詳細（`/consults/[id]`）

住人からの相談に回答するインタラクティブ画面。選択肢クリック→「回答する」で確定→住人の返答+信頼度変化表示。

- **home（`/home`）**: 相談詳細は中央ポップアップで表示
- **reports（`/reports`）**: 相談詳細は右側スライドインで表示
- **日報導線**: 未回答相談は日報に出さないため、詳細導線も発生しない

### オンボーディング画面（`/onboarding`）

初回起動時のセットアップフロー。`(onboarding)` ルートグループで、RoomStage（3D部屋）は読み込まない軽量レイアウト。背景は「部屋に入る前」をイメージしたダークブルー（`#0a1520`）。

単一ページ内でReact stateによるステップ管理を行う（URLベースだと戻るボタン問題が発生するため）。

**2つの入口パス:**
- **「新規で開始」**: 選択画面 → プラポリ同意 → Googleログイン → 名前入力 → `/home`（チュートリアルモード）
- **「ログインして続ける」**: 選択画面 → Googleログイン → `/home`（既存ユーザーなら通常モード）

| ステップ | 内容 | 完了条件 |
|---|---|---|
| 0 | 選択画面 | 「新規で開始」or「ログインして続ける」を選択 |
| 1 | プライバシーポリシー同意（新規パスのみ） | 「同意する」押下 → `privacy_accepted_at` 保存 |
| 2 | Googleログイン（新規パスのみ） | Google OAuth完了 |
| 3 | プレイヤー名入力 | 名前送信 → `player_name` 保存 → `/home` に遷移 |

住人登録はオンボーディング画面では行わず、ダッシュボード内のチュートリアルモードで管理室に誘導する。

#### チュートリアルモード（ホーム画面）

`onboarding_completed === false` かつ名前設定済みのユーザーがホーム画面を開くと、通常の3パネル（会話ログ・相談・みんなの様子）の代わりにチュートリアル案内パネルを表示:
- 「ようこそ、{player_name}さん」
- 「管理室から住人を2人以上登録して、世界を始めましょう。」
- 「管理室へ →」ボタン
- 登録済み住人数（「N人 / 2人」）

#### チュートリアルモード（管理室）

`/office/new` の住人登録で2人目が完了した時点で:
1. `onboarding_completed: true` を保存
2. `triggerConversationNow({ force: true })` で初回会話を即時生成
3. `/home` に遷移（通常モードに切り替わる）

#### リダイレクトロジック

- **ダッシュボード→オンボーディング**: `(dashboard)/layout.tsx` 内の `useOnboardingGuard` フックで、**未ログイン**、またはログイン済みだがプラポリ未同意・名前未設定の場合に `/onboarding` へリダイレクト
- **オンボーディング→ダッシュボード**: 既に `onboarding_completed === true` の場合は `/home` へリダイレクト
- **名前設定済み＆完了前**: ダッシュボードに留まる（チュートリアルモード）

#### スケジューラーゲーティング

会話・天気・日次減衰の3スケジューラーは `onboarding_completed === true` になるまで起動しない。オンボーディング中のAPI呼び出しを防止する。

**実装参考**: `apps/web/app/(onboarding)/onboarding/page.tsx`、`apps/web/lib/hooks/use-onboarding-guard.ts`、`apps/web/components/room/home-content.tsx`。

### その他

- プライバシーポリシー（`/legal/privacy`）: 実装済み
- 連絡フォーム（`/contact`）: 未実装

## 7.3 モーダル・オーバーレイUI

| UI | 内容 |
|---|---|
| 会話ログ詳細パネル | 右からスライドインするモーダル。全セリフ + SYSTEMメッセージ表示 |
| 相談詳細パネル（home） | 中央ポップアップ構造。選択肢UI + 回答確定 |

## 7.4 画面遷移

```
/ (リダイレクト) → /home
  └─ 未ログイン or onboarding未完了(名前未設定) → /onboarding
       → [新規] 選択 → 同意 → ログイン → 名前入力 → /home(チュートリアル)
       → [既存] 選択 → ログイン → /home(通常)
  └─ /home (チュートリアルモード: 住人<2人)
       → 管理室 → 住人登録×2 → onboarding完了 → /home(通常)

/home ──────→ /office（3D回転 right）
  │           ├─→ /office/residents（住人一覧）
  │           │    ├─→ /office/residents/[id]（住人詳細）
  │           │    │    ├─→ /office/residents/[id]/edit（住人編集）
  │           │    │    └─→ /office/relations/[id]（関係詳細）
  │           │    └─→ /office/new（新規住人登録）
  │           ├─→ /office/presets（プリセット管理）
  │           ├─→ /settings（設定）
  │           ├─→ /playguide（遊び方）
  │           ├─→ /specs（仕様）
  │           └─→ /legal/privacy（プライバシーポリシー）
  │
  ├──────→ /reports（3D回転 left・日報）
  │
  └── [通知タップ/カードタップ]
       ├─→ 会話ログ詳細（スライドインモーダル）
       └─→ 相談詳細（home:中央ポップアップ / reports:スライドイン / 直接遷移: /consults/[id]）
```
