# 03. 関係・感情系

---

## 3.1 関係

2住人間の「社会的関係」を6段階で分類するシステム。**対称・双方向で一つの値**を持つ（「AとBは友達」のように対称的）。

### 6段階の関係

| 関係 | コード値 | 親密度 | 知識伝播重み | 日本語ラベル |
|---|---|---|---|---|
| なし | `none` | 0 | 0.00 | なし |
| 認知 | `acquaintance` | 1 | 0.15 | 認知（顔見知り） |
| 友達 | `friend` | 3 | 0.45 | 友達 |
| 親友 | `best_friend` | 4 | 0.70 | 親友 |
| 恋人 | `lover` | 5 | 0.80 | 恋人 |
| 家族 | `family` | 4 | 0.65 | 家族 |

### 関係の特性

- **対称的**: `(aId, bId)` の正規化ペアで管理。「AはBの友達だがBはAの友達でない」は存在しない
- **ID正規化**: `aId` は常にUUID辞書順で小さい方。ユニーク制約 `(aId, bId)` で重複防止
- **「none = 会話不可」**: `none` のペアは会話パイプラインとスケジューラーの両方で除外される

### 各システムへの影響

| 参照先 | 影響 |
|---|---|
| 会話スケジューラー | `none` → 会話候補から除外 |
| 話題選定 | 親密度で話題スコアリング。低親密度は一部話題にペナルティ |
| 会話構造決定 | friend/best_friend/lover/family は warm 温度感の前提条件 |
| 知識伝播 | 伝播確率の50%を関係の親密度重みが占める。`none` は確率0 |
| 共有スニペット | 関係ごとの発生確率（none=0%, acquaintance=10%〜lover=60%） |
| LLMプロンプト | 「関係性: 友人」として会話設定セクションに注入 |

**実装参考**: `relations` テーブル（`apps/web/lib/drizzle/schema.ts`）。`relation_type` pgEnum。

## 3.2 印象

住人Aが住人Bに抱く感情のカテゴリ。**非対称・一方向**（A→BとB→Aは独立）。**通常系列**と**特別系列**の2系列がある。

### 通常系列（なし / 認知 / 友達 で使用）

```
嫌いかも ← なし → 気になる → 好きかも
```

| 順序 | ラベル | コード値 |
|---|---|---|
| 0 | 嫌いかも | `maybe_dislike` |
| 1 | なし | `none`（初期値） |
| 2 | 気になる | `curious` |
| 3 | 好きかも | `maybe_like` |

### 特別系列（親友 / 恋人 / 家族 で使用）

```
嫌い ← 普通 ← 好き（初期値）→ 大好き
```

| 順序 | ラベル | コード値 |
|---|---|---|
| 0 | 嫌い | `dislike` |
| 1 | 普通 | `none` |
| 2 | 好き | `like`（初期値） |
| 3 | 大好き | `love` |

### 系列間の切り替え

- **昇格時**（友達→親友/恋人）: 例外なくお互い「好き」（特別系列の初期値）にリセット
- **降格時**（親友/恋人→友達）: 好感度に基づく自動付与テーブルで通常系列の印象を決定
- 通常系列と特別系列の間に直接遷移はなく、**関係ラベルの変化により系列ごと切り替わる**

### 印象の変動

- 会話評価エンジンの `favor` 値に基づく
  - `favor ≥ +1` → 1段階上昇
  - `favor ≤ -1` → 1段階下降
  - `-1 < favor < +1` → 変化なし
- **1回の会話で最大1段階**しか動かない
- 両系列で共通の評価ロジックを適用

### 時間経過による変化

- 「気になる」: 7日間未接触で「なし」に戻る
- 「気まずい」（後述）: 7日間未接触で自動リセット

## 3.3 気まずい（awkward）— 特殊状態

通常の印象段階とは独立した特殊状態。元の印象を退避して付与され、時間経過で自然回復する。

### 二層構造

```
ImpressionState = {
  base:              印象の基本段階（通常/特別系列）
  special:           'awkward' | null
  baseBeforeSpecial: special付与前のbaseを退避
}
```

### ライフサイクル

1. **付与**: GPT出力の `setAwkward` フラグ → `base` を `baseBeforeSpecial` に退避、`special = 'awkward'`
2. **自然回復**: 次の会話評価時に **25%の確率** で `base = baseBeforeSpecial` に復元
3. **強制解除**: GPT出力の `clearAwkward` フラグ → 即座に復元
4. **永続化**: `special === 'awkward'` なら `feelings.label = 'awkward'`、そうでなければ `base` をマッピング

### 設計意図

恋人同士（`base: 'like'`）が喧嘩して `awkward` になっても、仲直りすれば元の `like` に戻れる。25%は「平均4回の会話で1回回復」を意味する。

**実装参考**: `packages/shared/types/conversation.ts`（`impressionStateSchema`）、`apps/web/lib/evaluation/evaluate-conversation.ts`（awkward処理）。

## 3.4 好感度

印象とは独立した0〜100の数値。感情の「現在地」を表す連続値。

### 基本仕様

| 項目 | 値 |
|---|---|
| 範囲 | 0〜100（整数） |
| 中立点 | **30** |
| UI表示 | 30が中央に来るよう擬似的に表示。**数値は非表示**、バーの色と長さで表現 |

### 初期値テーブル

関係ラベルに応じた自動提案値:

| 関係 | 初期値 |
|---|---|
| なし | 30 |
| 認知 | 35 |
| 友達 | 45 |
| 親友 | 65 |
| 恋人 | 65 |
| 家族 | 55 |

### 変動

会話評価エンジンの `favor` デルタ（-2〜+2）が `Math.round()` されて加算される。1回の会話でスコアは**最大2**しか動かない。

### 非対称化

現行の対称的な変動（A→BとB→Aに同額加算）に対し、非対称性を導入するかどうかを含めて検討中。

> ※ 非対称化の必要性・実現手段は別途議論予定

### 自然減少

好感度は中立点30に向かって**時間経過で収束**する。

| 最終接触からの経過 | 日あたりの減少 |
|---|---|
| 3日以内 | 0（変化なし） |
| 4〜7日 | -1/日（30を下回る場合は+1/日） |
| 8日以上 | -2/日（30を下回る場合は+2/日） |

**実装参考**: `feelings` テーブル（`apps/web/lib/drizzle/schema.ts`）。`fromId`, `toId`, `label`（feeling_label pgEnum）, `score`（integer）。

## 3.5 好感度変動ロジック（会話評価エンジン）

GPTが生成した会話結果を入力として、好感度デルタ（favor）・印象変化・スレッド進行状態を**ルールベースで算出**するシステム。

### 評価パイプライン

```
① タグ重み加算
   ポジティブ: 共感+0.6, 感謝+0.7, 称賛+0.8, 協力+0.5, 軽い冗談+0.2, 雑談・共通+0.1
   ネガティブ: 否定-0.8, 皮肉-0.5, 非難-1.2
      ↓
② 会話バランスボーナス
   完全均衡で最大+0.2、完全偏りで+0.0
      ↓
③ 品質ヒント加点
   coherence.good +0.2 / coherence.poor -0.4
   tone.gentle +0.2 / tone.harsh -0.4
      ↓
④ シグナルによるスレッド進行
   signals合計 ≥ 0.2 → done / > 0 → ongoing / ≤ 0 → paused
      ↓
⑤ favor クリップ [-2, +2]
      ↓
⑥ 印象1段階移動（favor ≥ +1 で上昇、≤ -1 で下降）
      ↓
⑦ special状態処理（awkward の付与/解除/25%自然回復）
```

### 対称性と非対称化

- 現行: タグ重みは A→B と B→A に**同額**加算
- 非対称化の必要性・実現手段は別途議論予定

### 設計上の特性

- **ネガティブの方が影響大**: 非難（-1.2）は単独で印象下降トリガー。称賛（+0.8）+バランス（+0.2）でようやく上昇トリガー
- **重み設定の外部化**: `public/config/conversation-weights.json` から重みを読み込む機構あり
- **決定論的**: awkardの自然回復（25%確率）を除き、同じ入力に対して同じ出力

**実装参考**: `apps/web/lib/evaluation/evaluate-conversation.ts`、`apps/web/lib/evaluation/weights.ts`。

## 3.6 信頼度

住人がプレイヤーに対して抱く信頼度。キャラ間の好感度とは**完全に別系統**。

### 基本仕様

| 項目 | 値 |
|---|---|
| 範囲 | 0〜100（整数） |
| 初期値 | 性格に応じたランダム値 |
| 保存先 | `residents.trust_to_player` |
| UI編集 | **不可**（管理室のフォームからは編集できない設計が正しい） |

### 初期値の算出

基準値50に、性格パラメータ5軸・MBTI2軸・ランダムジッターを加算して決定する。

```
trustToPlayer = clamp(0, 100,
  50
  + (sociability - 3) × 2
  + (empathy - 3) × 1.5
  + (expressiveness - 3) × 1
  + (activity - 3) × 0.5
  + (stubbornness - 3) × -1
  + mbtiBonus
  + jitter
)
```

**性格パラメータ補正**（各パラメータは1〜5、中央3で±0）:

| パラメータ | 係数 | 範囲 | 根拠 |
|---|---|---|---|
| sociability | ×2 | -4〜+4 | 社交的な人ほど他者を信頼しやすい |
| empathy | ×1.5 | -3〜+3 | 気配りできる人は関係構築に前向き |
| expressiveness | ×1 | -2〜+2 | 表現豊かな人はオープンな態度 |
| activity | ×0.5 | -1〜+1 | 行動力はやや関連 |
| stubbornness | ×-1 | -2〜+2 | 頑固な人は警戒心が強い |

**MBTI補正**:

| 軸 | 補正 | 根拠 |
|---|---|---|
| E（外向） | +2 | 外向的で対人関係にオープン |
| I（内向） | -2 | 内向的で慎重 |
| F（感情） | +3 | 感情重視で共感的、信頼を置きやすい |
| T（思考） | -3 | 論理重視で慎重に判断 |

> ※ S/N軸・J/P軸は信頼度への影響が小さいため補正対象外

**ジッター**: ±2（整数、一様乱数）

**算出例**:

| ケース | 計算 | 初期値 |
|---|---|---|
| 全パラメータ3, MBTI未設定 | 50 ± 2 | 48〜52 |
| sociability=5, empathy=4, ENFP | 50 + 4 + 1.5 + 0 + 0 + 0 + 2 + 3 ± 2 | 59〜63 |
| sociability=1, stubbornness=5, ISTJ | 50 - 4 + 0 + 0 + 0 - 2 - 2 - 3 ± 2 | 37〜41 |

> ※ 最終値は `Math.round()` で整数化し、0〜100にクランプ

### 変動経路

**相談（consult）の回答評価**が主な変動経路。選択肢ごとに定義された `trustDeltaByChoice` に基づいて変動する。

### 信頼度の影響

| 影響先 | 内容 |
|---|---|
| 相談の発生頻度 | 信頼度が高いほど頻繁に相談が来る |
| 相談の深さ | 信頼度が高いほど深い内容の相談 |
| 態度 | 4段階: 警戒 → 距離あり → 好意的 → 親密 |

### バグ修正予定

- `trustToPlayer` がUI上で編集可能になっている（本来は編集不可が正しい設計）

## 3.7 関係ラベル遷移

関係ラベルの変化は、**観察型**（プレイヤー関与不要）と**介入型**（プレイヤーが相談に回答）の2種に分かれる。

### 遷移図

```
なし ──(確率/挨拶)──→ 認知 ──(イベント)──→ 友達 ─┬─(イベント)──→ 親友
                                                  └─(告白相談)──→ 恋人
家族 ─── 変化なし（初期設定 / 管理室でのみ設定）

降格: 親友/恋人 ──(別れ相談)──→ 友達
```

### なし → 認知（観察型）

| 項目 | 仕様 |
|---|---|
| 発生方法 | ① 時間経過の確率抽選（一定時間ごとに `none` のペアを対象）、または ② 「挨拶」Topic（`none` 専用の会話） |
| プレイヤー関与 | 不要 |

### 認知 → 友達（観察型）

| 項目 | 仕様 |
|---|---|
| 発生形式 | 「友達になる」イベント（通常会話と同じ形式で生成・表示） |
| 発生確率 | 好感度が高いほど UP |
| ブロック条件 | 好感度20以下 or 印象「嫌いかも」→ 発生しない |
| プレイヤー関与 | 不要 |

### 友達 → 親友（観察型）

| 項目 | 仕様 |
|---|---|
| 発生形式 | 「親友になる」イベント |
| 必須条件 | 双方の印象が「好きかも」＋ 平均好感度60以上 |
| 発生確率 | 好感度が高いほど UP |
| プレイヤー関与 | 不要 |

### 友達 → 恋人（介入型：告白）

| 項目 | 仕様 |
|---|---|
| 発生形式 | 告白相談 — キャラがプレイヤーに相談 |
| 発生条件 | 片方の印象が「好きかも」＋ 好感度しきい値（「要調整。前身では70、下げる方向」） |
| プレイヤーの選択 | アドバイス or 止める（止めた場合は告白自体が発生しない） |
| 成功率 | 3因子で算出（下記参照）。クランプなし — 0%〜100%の全範囲 |
| 成功時 | 関係 → 恋人、印象 → 双方「好き」にリセット |
| 失敗時 | 関係変化なし、印象が「気まずい」に |

#### 告白成功率の算出

```
P(成功) = favorFactor × 0.5 + impressionFactor × 0.3 + personalityFactor × 0.2
```

| 因子 | 算出方法 | 範囲 |
|---|---|---|
| favorFactor | `max(0, (相手の好感度 - 30) / 70)` | 0.0〜1.0 |
| impressionFactor | 相手の印象段階による（下表） | 0.0〜1.0 |
| personalityFactor | `(相手のempathy - 1) / 4` | 0.0〜1.0 |

**印象ごとのimpressionFactor**:

| 印象（相手 → 告白者） | 値 |
|---|---|
| 好きかも | 1.0 |
| 気になる | 0.6 |
| なし | 0.3 |
| 嫌いかも | 0.0 |

- 重み合計が1.0のため、結果は自然に0%〜100%の範囲に収まる（人為的なクランプは不要）
- 極端な関係値であれば確実な成功/失敗になることが自然
- **相手視点**: 成功率は告白される側の好感度・印象・empathyで決まる

### 降格：親友/恋人 → 友達（介入型：別れ）

| 項目 | 仕様 |
|---|---|
| 発生形式 | 別れ相談 — キャラがプレイヤーに相談 |
| 発生条件 | 印象「嫌い」かつ好感度15以下 |
| プレイヤーの選択 | 回答次第で降格が確定 |
| 降格時 | 印象は特別系列 → 通常系列に変換（好感度ベースの自動付与テーブル使用） |

### 家族

- 遷移なし。初期設定 or 管理室でのみ設定
- 種別情報（兄/姉/父/母など）付き
- 親友・恋人とは排他（家族は他ラベルと併存不可）

### 手動設定

プレイヤーはキャラ追加時などに初期関係を設定できる（元々仲の良い人物の登録に対応）。メインコンテンツは `none` から始まって会話・イベントを通じて関係が変化していく流れ。

```
                    └─────┬──────┬──────┬───────┘
                          │      │      │
                          ▼      ▼      ▼
              ┌──────┐ ┌──────┐ ┌─────────────┐
              │関係   │ │好感度│ │信頼度        │
              │      │ │+印象 │ │(trustToPlayer)│
              └──┬───┘ └──┬───┘ └──────┬──────┘
                 │        │             │
   ┌─────────────┼────────┘             │ 相談
   │             │                      │ イベント
   ▼             ▼                      ▼ のみ変動
┌──────────────────────────┐    ┌──────────────┐
│  会話パイプライン          │    │ 相談システム    │
│                          │    │ (consult)     │
│  ① 会話可否判定(type≠none)│    └──────────────┘
│  ② 話題選定(親密度,好感度) │

│  ⑤ GPT呼び出し           │
└──────────┬───────────────┘
           │ GPT出力
           ▼
┌──────────────────────────┐
│  会話評価エンジン          │
│  → favor → 好感度更新     │
│  → impression → 印象更新  │
│  → awkward管理            │
└──────────────────────────┘
```
