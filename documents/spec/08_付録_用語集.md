# 08. 付録 — 用語集

---

## キャラクター関連

| 用語 | 定義 |
|---|---|
| **住人** | ゲームに登場するキャラクター |
| **プレイヤー** | ゲームの管理人。住人を観察・介入する上位存在 |
| **MBTI** | 16タイプの性格類型。住人の「認知スタイル」（何を考えるか、どう判断するか）を表す |
| **性格パラメータ** | 社交性・気配り・頑固さ・行動力・表現力の5軸スライダー（各1〜5）。住人の「行動パターン」（どう振る舞うか）を表す |
| **話し方プリセット** | 語尾・口癖・禁止表現・例文を定義したデータ。LLMプロンプトの最重要ルールとして使用される |
| **一人称** | 住人が自分を指す言葉（「俺」「私」など）。話し方プリセットとは独立に管理 |
| **呼び方** | 住人Aが住人Bを呼ぶ呼び名（ニックネーム）。方向性あり（A→BとB→Aは別） |

## 関係・感情関連

| 用語 | 定義 |
|---|---|
| **関係** | 2住人間の社会的関係。対称・双方向で一つの値を持つ。none（なし）/ acquaintance（認知）/ friend（友達）/ best_friend（親友）/ lover（恋人）/ family（家族）の6段階 |
| **印象** | 住人Aが住人Bに抱く感情のカテゴリ。非対称・一方向。**通常系列**（嫌いかも/なし/気になる/好きかも）と**特別系列**（嫌い/普通/好き/大好き）の2系列がある |
| **好感度** | 印象とは独立した0〜100の数値（中立点30）。感情の現在地を表す連続値。UI上では数値非表示、バーの色と長さで表現 |
| **信頼度** | 住人がプレイヤーに対して抱く信頼度（0〜100）。相談を通じて変動する |
| **気まずい（awkward）** | 通常の印象段階とは独立した特殊状態。元の印象を退避して付与され、時間経過で自然回復する |

## 会話関連

| 用語 | 定義 |
|---|---|
| **会話スレッド** | 同一ペアの会話の文脈的なまとまり。ongoing（進行中）/ paused（中断）/ done（完了）のステータスを持つ |
| **会話記憶** | 前回の会話の要約・話題・未解決スレッドを構造化したデータ。次回の会話プロンプトに注入され、文脈の連続性を保つ |
| **話題ソース** | 会話の話題がどこから来たかの分類。shared_interest / personal_interest / continuation / snippet / third_party / self_experience / heart_to_heart / small_talk / seasonal の9種 |
| **相談（consult）** | 住人がプレイヤーに持ちかけるインタラクション。選択肢を選ぶことで信頼度が変化する。日常相談と関係遷移相談（告白・別れ）の2種がある |
| **スタンス** | 会話における各キャラクターの態度。enthusiastic（乗り気）/ agreeable（穏やか）/ reluctant（面倒）/ indifferent（興味薄）/ confrontational（突っかかり）の5種 |
| **温度感** | 会話全体の雰囲気。warm（和やか）/ lukewarm（ぎこちない）/ neutral（普通）/ tense（緊張）の4種 |

## 世界関連

| 用語 | 定義 |
|---|---|
| **共有スニペット** | 住人ペアの「偶然の共有体験」を表すテキスト。場所×状況×修飾語の組み合わせで生成される |
| **最近の出来事** | 住人に最近起きた日常的な出来事。GPTで生成され、知識伝播の素材になる |
| **知識伝播** | 住人の出来事が他の住人に確率的に伝わる仕組み。伝播確率は関係の親密度と情報源の性格に依存する |
| **睡眠スケジュール** | 各住人の就寝・起床時刻。毎日±60分のランダム化あり。active（活動中）/ preparing（就寝準備中）/ sleeping（睡眠中）の3状態 |

## UI関連

| 用語 | 定義 |
|---|---|
| **ホーム** | メインのダッシュボード画面。住人の「今」を一覧する。3D部屋の正面（front） |
| **管理室（Office）** | 住人やプリセットの管理を行う画面群。3D部屋の右面（right） |
| **日報（Reports）** | 過去の会話・相談イベントを時系列で閲覧する画面。3D部屋の左面（left） |
| **覗く** | プレイヤーが任意の住人の「今の様子」を能動的に見る機能。LLMで生成 |

## 技術用語の対応表

| 仕様書での用語 | コード上の表現 | 保存先 |
|---|---|---|
| **関係** | `RelationType` | `relations` テーブル `type` カラム |
| **印象** | `FeelingLabel` / `ImpressionState` | `feelings` テーブル `label` カラム / 評価エンジン内部の `ImpressionState`（base + special） |
| **好感度** | `Feeling.score` | `feelings` テーブル `score` カラム（integer 0〜100） |
| **信頼度** | `trustToPlayer` | `residents` テーブル `trust_to_player` カラム（integer 0〜100） |

## イベント型の分類

| 型 | 説明 |
|---|---|
| **観察型** | プレイヤー関与なしに関係が自然に変化するイベント（友達になる、親友になるなど） |
| **介入型** | 住人からの相談にプレイヤーが回答することで関係遷移に関与するイベント（告白、別れなど） |
