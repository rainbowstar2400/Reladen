# 02. キャラクター定義系

---

## 2.1 キャラクタープロフィール

住人の全属性を統合管理するマスターエンティティ。性格（MBTI＋性格パラメータ）・人口統計（年齢・性別等）・表現スタイル（話し方・一人称）・生活リズム（睡眠）を一元化し、全ゲームシステムの基盤データを提供する。

### データ構造

| フィールド | 型 | 制約 | 説明 |
|---|---|---|---|
| id | UUID | PK | 一意識別子 |
| name | string | 必須, 1文字以上 | 表示名 |
| mbti | MbtiEnum | 任意 | 16タイプのいずれか |
| traits | object | 各1-5, 全項目任意 | 性格パラメータ（5軸） |
| speechPreset | UUID | 任意, FK→presets | 話し方プリセットへの参照 |
| trustToPlayer | integer | 0-100 | プレイヤーへの信頼度（※UI編集不可） |
| gender | enum | 任意 | male / female / nonbinary / other |
| age | integer | 0-120, 任意 | 年齢 |
| birthday | string | 任意, "MM/DD" | 誕生日 |
| occupation | UUID | 任意, FK→presets | 職業プリセットへの参照 |
| firstPerson | UUID | 任意, FK→presets | 一人称プリセットへの参照 |
| interests | string[] | 最大20個 | 趣味・興味タグ |
| sleepProfile | SleepProfile | 任意 | 睡眠スケジュール |

**実装参考**: `residents` テーブル（`apps/web/lib/drizzle/schema.ts`）。`traits`, `interests`, `sleepProfile` は JSONB。`speechPreset`, `occupation`, `firstPerson` は `presets.id` への FK（`onDelete: 'set null'`）。

### プリセット参照パターン

`speechPreset`・`occupation`・`firstPerson` はすべてUUIDで `presets` テーブルを参照する統一設計。プリセットは `category`（`'speech'` / `'occupation'` / `'first_person'`）で区別される。プリセット削除時は住人のフィールドが `null` に設定される（`onDelete: 'set null'`）。

### 名前の仕様

- 苗字・名前を区別しない（単一テキスト。プレイヤーに委ねる）
- 名前はゲーム内で一意であることを推奨するが、制約はかけない

## 2.2 MBTI

16タイプのMBTI性格類型で住人の**認知スタイル**（何を考えるか、どう判断するか）を分類するシステム。

### 判定方法

- 4軸（E/I, S/N, T/F, J/P）× 各2問 = 計8問
- 回答スコア（1-5）を `score - 3` で -2〜+2 に変換し、`direction` を乗じて軸ごとに加算
- 各軸の合計が `≥ 0` なら先頭文字（E/S/T/J）、`< 0` なら後方文字（I/N/F/P）
- 4文字を結合して MbtiType を確定

### 役割分担

MBTIは**認知**を担い、性格パラメータは**行動**を担う。

- MBTI → LLMプロンプトの「性格傾向」として参考情報として注入
- MBTIはドミナンススコアや感情スタンスの算出には**使用されない**

**実装参考**: `packages/shared/logic/mbti.ts`（判定ロジック）、`packages/shared/gpt/prompts/trait-descriptions.ts`（16タイプの日本語説明マッピング）。

## 2.3 性格パラメータ（5軸スライダー）

住人の**行動表出パターン**（どう振る舞うか）を5軸のスライダーで定義するシステム。会話構造の決定ロジック・知識伝播確率・LLMプロンプトの行動記述に直接影響する。

### 5つの軸

| 軸 | 説明 | 範囲 | デフォルト |
|---|---|---|---|
| 社交性（sociability） | 会話への積極性 | 1-5 | 3 |
| 気配り（empathy） | 相手の気持ちへの配慮 | 1-5 | 3 |
| 頑固さ（stubbornness） | 意見への固執度 | 1-5 | 3 |
| 行動力（activity） | 行動提案への積極性 | 1-5 | 3 |
| 表現力（expressiveness） | 感情表現の豊かさ | 1-5 | 3 |

### 3段階ティア変換

| 値 | ティア | 例（社交性） |
|---|---|---|
| 1-2 | low | 会話は短く済ませたがり、沈黙を嫌がらない |
| 3 | neutral | （LLMプロンプトでは省略） |
| 4-5 | high | 自分から話題を振り、沈黙が続くと自ら埋めようとする |

各特性×3段階 = 計15パターンの日本語行動記述が定義されている。neutral（3）の場合はLLMプロンプトから省略し、トークンを節約する。全5特性が中立なら `"行動特性: 特筆すべき偏りなし（平均的）"` の1行で済む。

### 他システムへの影響

| 参照先 | 影響 |
|---|---|
| ドミナンススコア | `sociability×0.35 + activity×0.3 + expressiveness×0.25 + stubbornness×0.1` |
| 感情スタンス決定 | 各特性の高低が5種のスタンスへの傾きに影響 |
| 知識伝播確率 | `sociability` と `expressiveness` が確率に影響 |
| 話題選定 | `sociability` / `expressiveness` がスコア補正に影響 |

**実装参考**: `packages/shared/types/conversation-generation.ts`（`traitsSchema`）、`packages/shared/gpt/prompts/trait-descriptions.ts`（行動記述マップ）。

## 2.4 話し方プリセット

住人の発話スタイル（語尾・口癖・禁止表現・例文）を構造化データとして管理し、LLMプロンプトの**最重要ルール**として使用するシステム。

### データ構造

| フィールド | 型 | 必須 | 説明 |
|---|---|---|---|
| label | string | ○ | プリセット名（例: "ギャル語"） |
| description | string | ○ | 説明文 |
| endings | string[] | ○（1個以上） | 語尾パターン（例: ["〜じゃん", "〜だろ"]） |
| frequentPhrases | string[] | | 口癖（例: ["マジで", "てかさ"]） |
| avoidedPhrases | string[] | | 禁止表現（例: ["敬語", "難しい言葉"]） |
| examples | string[] | ○（1個以上） | セリフ例 |

### LLMプロンプトへの注入

話し方プリセットはプロンプトの**最先頭**（【最重要: 口調ルール】セクション）に配置され、primacy biasを活用する。さらに末尾の【生成ルール】でも再確認指示を出す（recency bias活用）。

```
■ ハル:
  一人称: 「俺」（厳守・表記揺れ禁止・自分の名前で自己言及しない）
  語尾: 〜じゃん / 〜だろ
  よく使う表現: マジで, てかさ
  避ける表現: 敬語, 丁寧語
  セリフ例: 「これマジでヤバいって。」
```

### LLM自動抽出

プリセットの `label` + `description` + `examples` をgpt-5-mini（`temperature: 0.3`）に入力し、構造化データを自動抽出する。抽出結果は `presets.speech_profile_data`（JSONB）にキャッシュされ、2回目以降はLLM呼び出し不要。

### 未設定時のフォールバック

話し方プリセットが未設定の場合は `"話し方: 未設定（自然な話し方で生成）"` がプロンプトに出力され、LLMに自然な口調で生成させる。

**実装参考**: `packages/shared/types/conversation-generation.ts`（`speechProfileSchema`）、`packages/shared/gpt/prompts/conversation-prompt.ts`（`formatSpeechRuleBlock()`）。

## 2.5 一人称

住人が自分を指す言葉。話し方プリセットとは独立に管理される。

- `presets` テーブルの `category='first_person'` レコードとして保存
- プロンプトでは「**厳守・表記揺れ禁止・自分の名前で自己言及しない**」という最強の制約がかかる
- バリデーションで表記揺れを検出（例: `"俺"` 指定時に `"おれ"` `"オレ"` が出現していないか検証）

### 表記揺れ検出パターン（一部）

| 一人称 | 禁止バリエーション |
|---|---|
| 私 | わたし, ワタシ, あたし, アタシ, わたくし |
| 俺 | おれ, オレ |
| 僕 | ぼく, ボク |

**実装参考**: `packages/shared/logic/conversation-validator.ts`（`FIRST_PERSON_VARIANTS`）。

## 2.6 呼び方（ニックネーム）

住人Aが住人Bを呼ぶ呼び名。**方向性あり**（A→BとB→Aは別レコード）。

### 現行の仕様

- `nicknames` テーブルで管理。ユニーク制約 `(fromId, toId)` で1ペア1方向につき1レコード
- プレイヤーが管理室（Office）で手動設定
- UI上の関係表示に使用

### 自動生成システム（SectionBの決定事項）

| 項目 | 仕様 |
|---|---|
| 初回生成 | 関係が none → acquaintance 時にLLMで生成 |
| 再生成 | 関係昇格時により親しい呼び方に再生成 |
| ロック機能 | プレイヤー手動設定の呼び方は「変化を許容するか」のロックフラグ |
| 傾向プリセット | キャラごとに呼び方の傾向を選択: あだ名 / 呼び捨て / さん付け / くん・ちゃん付け / 上下重視 |
| プロンプト注入 | 生成された呼び方を会話プロンプトに注入（現行の未注入を解消） |

### 設計詳細（設計決定事項）

#### スキーマ変更

- `residents` テーブルに `nickname_tendency` カラム追加（enum: `nickname` / `bare` / `san` / `kun_chan` / `hierarchy`）
- `nicknames` テーブルに `locked` カラム追加（`BOOLEAN DEFAULT false`）

#### 生成トリガーとLLMへの指示

| 遷移 | 生成 | LLMへの指示方針 |
|------|------|----------------|
| none → acquaintance | する | 傾向プリセット重視: 「このキャラの傾向は〇〇。顔見知りレベルの呼び方を生成して」 |
| acquaintance → friend | しない | — |
| friend → best_friend / lover | する | 関係レベル重視: 「この2人は親友/恋人になった。今の呼び方は〇〇。より親密な呼び方に変えて」 |

- 初回生成（none → acquaintance）: 傾向プリセットに従った呼び方を生成
- 再生成（friend → best_friend/lover）: 傾向プリセットは参考程度。現在の呼び方 + 新しい関係レベルを主な入力とし、LLMが文脈から判断
- LLMには双方向（A→BとB→A）を1回のコールで生成させる

#### ロック機構

- プレイヤーがOfficeで手動設定 → 自動的に `locked = true`
- `locked = true` の呼び方は自動再生成の対象外（スキップ）
- UIにトグルを設置し、プレイヤーが明示的にロック解除（`locked = false` に戻す）可能
- 手動変更時にトグルが `true` に切り替わったことがUIで視覚的に分かるようにする

#### プロンプト注入

会話生成時に「AはBを〇〇と呼ぶ」「BはAを〇〇と呼ぶ」を会話プロンプトの人物情報に含める。

**実装参考**: `apps/web/lib/drizzle/schema.ts`（`nicknames` テーブル）、`apps/web/lib/data/nicknames.ts`（CRUD操作）。

## 2.7 システム間の関係

```
┌─────────────┐     ┌──────────────┐
│   MBTI      │     │ 性格パラメータ │
│ (認知スタイル) │     │ (行動パターン) │
└──────┬──────┘     └──────┬───────┘
       │                    │
       │  LLMプロンプト      │  ロジック直接参照
       │  (参考情報)         │  (ドミナンス/スタンス/伝播)
       ▼                    ▼
┌──────────────────────────────────┐
│     キャラクタープロフィール        │ ← マスターエンティティ
│     (residents テーブル)          │
│  ┌─ mbti, traits ────────────┐  │
│  │ speechPreset → presets FK  │  │
│  │ firstPerson → presets FK   │  │
│  │ occupation  → presets FK   │  │
│  └────────────────────────────┘  │
└────────┬──────────┬──────────────┘
         │          │
         ▼          ▼
┌────────────┐  ┌──────────────┐
│ 話し方      │  │ 呼び方        │
│ プリセット   │  │ (ニックネーム)  │
│ ↓           │  │              │
│ LLMプロンプト │  │ LLMプロンプト  │
│ 最重要ルール  │  │ に注入        │
└────────────┘  └──────────────┘
```
