# 第3章 ドメインモデル
- 共通項目: `id(uuid)` / `updated_at` / `deleted` / `owner_id`(RLS)。tombstoneの意味と状態遷移。
- 住人(Resident): 名前/性別/性格/口調/呼称設定(呼び捨て/さん付け等)/MBTI回答/睡眠プロファイル/所属等。Presetの扱い。
- 関係段階(Relation): 認知→友達→親友→恋人(例)。段階ごとの閾値や必要イベント、呼称のデフォルト変更。TempRelationDataの補足。
- 会話(Conversation): 話者/対象/トーン/生成元(GPT/スクリプト)/発生条件/状態(公開/新聞/掲示板転載)/メタ情報(気分/天気反映)。
- トピック(Topic): WorldFactsから派生、カテゴリ/優先度/期限/関連住人。消化状態管理。
- WorldFacts: 世界設定の断片。生成ルール、Topic紐付け、保持期間。
- 天気(Weather): 種類・更新周期、住人/会話/新聞への反映。
- イベント(Event): 誕生日、寝言、会話抽選、掲示板発火、実績解除などのトリガー・条件・効果。
- 実績(Achievement): 条件(関係段階達成/会話回数/天気関連)、報酬表示、解除状態。
- 新聞/掲示板: 表示対象、スレ/記事単位のデータ構造、公開範囲、多人数会話の扱い。
